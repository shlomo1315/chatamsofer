# מפרט צד־שרת — טופס רישום איגוד צאצאי החתם סופר

מסמך זה מגדיר את כל נקודות הקצה (endpoints) של השרת עבור טופס הרישום שנבנה בממשק נדרים פלוס ("מחלקת טפסים").

---

## כללי

- **כתובת בסיס:** `BASE_URL = https://<דומיין-האיגוד>/api/nedarim-form`
  (הדומיין המדויק יימסר בעת העלייה; מבנה הנתיבים סופי.)
- **פורמט:** JSON בלבד. קריאות GET/POST רגילות.
- **CORS:** כל נקודות הקצה מחזירות `Access-Control-Allow-Origin: https://matara.pro` ומטפלות ב-preflight (בקשת `OPTIONS`).
- **מטרת הטופס:** רישום בלבד. אין הגשת בקשות (בקשה מחייבת העלאת קבצים — לא נתמך בנדרים).

---

## סקירת נקודות הקצה

| # | Method | Endpoint | תיאור |
|---|--------|----------|-------|
| 1 | GET | `/lineage-roots` | הדור הראשון (החתם סופר) |
| 2 | GET | `/lineage-children?nodeId=<id>` | צאצאים ישירים של צומת |
| 3 | GET | `/lookup?id=<ת"ז>` | בדיקה אם הת"ז כבר רשום |
| 4 | GET | `/cities` | רשימת ערים |
| 5 | GET | `/streets?city=<עיר>` | רחובות של עיר |
| 6 | POST | `/verify/send` | שליחת קוד אימות לטלפון |
| 7 | POST | `/verify/confirm` | אימות הקוד → החזרת טוקן |
| 8 | POST | `/register` | שמירת הרישום הסופי |

---

## 1. `GET /lineage-roots`

מחזיר את הדור הראשון (מרן החתם סופר זי"ע). קריאה זו מספקת את ה-`nodeId` של השורש — **אין להטמיע מזהה קשיח**; יש לקבלו מכאן.

**תגובה:**
```json
{
  "children": [
    { "nodeId": "root-uuid", "name": "מרן החתם סופר זי\"ע", "relation": null }
  ]
}
```

---

## 2. `GET /lineage-children?nodeId=<id>`

מחזיר את הצאצאים הישירים של הצומת. זו הקריאה המרכזית בבורר הדורות — נקראת בכל בחירת דור.

**תגובה:**
```json
{
  "children": [
    { "nodeId": "abc-123", "name": "רבי אברהם שמואל בנימין", "relation": "son" },
    { "nodeId": "abc-124", "name": "מרת הינדל אשת רבי דוד צבי ארנפלד", "relation": "son_in_law" }
  ]
}
```

**שדות:**
- `nodeId` — מזהה ייחודי קבוע של הצומת. זהו הערך שיישלח בשמירה הסופית.
- `name` — השם המלא להצגה.
- `relation` — `"son"` (בן) / `"son_in_law"` (חתן) / `null`.

**דור אחרון:** אם לצומת אין צאצאים → `{ "children": [] }` (מערך ריק). כך הטופס יודע לעצור ולהציע למשתמש להוסיף את עצמו.

---

## 3. `GET /lookup?id=<תעודת_זהות>`

בדיקה אם תעודת הזהות כבר רשומה במערכת. **יש לקרוא לפני מעבר להרשמה.**

**תגובה — צאצא רשום בעצמו (הבעל/הרשום או בן/בת הזוג):**
```json
{
  "found": true,
  "message": "תעודת הזהות כבר רשומה במערכת האיגוד. לא ניתן להגיש בקשות דרך נדרים פלוס כיוון שכל בקשה מחייבת העלאת קבצים. להגשת בקשות יש לפנות במייל: igud@chasamsofer.info"
}
```
→ יש להציג את `message` כמות שהוא ולעצור. **אין לעבור להרשמה.** (אין שדה `is_child`.)

**תגובה — ילד של צאצא (קיים בתוך `children` של רשומה אחרת):**
```json
{
  "found": true,
  "is_child": true,
  "parent_name": "שם ההורה הצאצא",
  "lineage_chain": [
    { "generation": 1, "name": "...", "relation": null },
    { "generation": 2, "name": "...", "relation": "son" }
  ],
  "child": { "name": "...", "id_number": "...", "birth_date": "...", "gender": "male|female" }
}
```
→ יש לנתב ל**רישום מהיר** (ולא לחסום). ההבחנה: הנוכחות של `is_child: true`.
`parent_name` ו-`lineage_chain` מיועדים לשיוך אוטומטי בסדר הדורות ולמילוי מראש;
`child` הוא נתוני הילד לנוחות (אופציונלי לשימוש).

**תגובה — הת"ז לא קיים:**
```json
{ "found": false }
```
→ מעבר לטופס ההרשמה הרגיל.

## 4. `GET /cities`

רשימת ערים לבחירה.

**תגובה:**
```json
{ "cities": ["ירושלים", "בני ברק", "..."] }
```

---

## 5. `GET /streets?city=<שם_עיר>`

רחובות של העיר הנבחרת. בחירת עיר טוענת את הרחובות שלה.

**תגובה:**
```json
{ "streets": ["רחוב א'", "רחוב ב'", "..."] }
```

---

## 6. `POST /verify/send`

שליחת קוד אימות לטלפון באמצעות שיחה קולית המקריאה את הקוד. **אימות טלפון הוא תנאי חובה לשמירת הרישום.**

**בקשה:**
```json
{ "channel": "phone", "value": "0501234567" }
```

**תגובה:**
```json
{ "ok": true, "sent": true }
```

---

## 7. `POST /verify/confirm`

אימות הקוד שהתקבל. בהצלחה מחזיר טוקן חתום (תקף 30 דקות) שיש לצרף בשמירה הסופית.

**בקשה:**
```json
{ "channel": "phone", "value": "0501234567", "code": "1234" }
```

**תגובה — הצלחה:**
```json
{ "ok": true, "token": "<טוקן>" }
```

**תגובה — כשל:**
```json
{ "error": "קוד שגוי. נסו שוב." }
```
(status `400`)

> **הערה על מייל:** אין אימות מייל בטופס זה. המשתמש מזין כתובת מייל רגילה; היא נכנסת למערכת כלא-מאומתת ומסומנת ככזו. **אין צורך במסך אימות מייל.** (אם בעתיד ירצה לקבל קוד למייל בכניסה לפורטל — יעבור אימות אז.)

---

## 8. `POST /register`

השמירה הסופית. שולח את כל פרטי הרישום + שרשרת הדורות שנבחרה + טוקן אימות הטלפון.

**בקשה:**
```json
{
  "id_number": "123456789",
  "id_doc_type": "id",
  "full_name": "שם פרטי",
  "family_name": "שם משפחה",
  "phone": "0501234567",
  "phone2": "",
  "email": "user@example.com",
  "city": "ירושלים",
  "address": "רחוב הרב X 12",
  "birth_date": "1990-01-01",
  "gender": "male",
  "marital_status": "נשואים",
  "spouse_name": "",
  "spouse_id_number": "",
  "spouse_phone": "",
  "children": [
    { "name": "", "id_number": "", "gender": "", "birth_date": "" }
  ],
  "notes": "",
  "phone_tokens": [
    { "value": "0501234567", "token": "<טוקן מ-verify/confirm>" }
  ],
  "lineage_chain": [
    { "generation": 1, "nodeId": "root-id", "name": "מרן החתם סופר זי\"ע", "relation": null },
    { "generation": 2, "nodeId": "abc-124", "name": "מרת הינדל...", "relation": "son_in_law" }
  ],
  "lineage_new_nodes": [
    { "afterGeneration": 2, "name": "שם שהוזן ידנית", "relation": "son" }
  ]
}
```

**שדות חובה:**
- `id_number`, `full_name`, `family_name`
- לפחות טלפון אחד מאומת (`phone_tokens`)
- `city` + `address` מלאה (רחוב ומספר בית)

**שדות דורות:**
- `lineage_chain` — השרשרת שנבחרה מהשורש ועד הנרשם (כל איבר: `generation`, `nodeId`, `name`, `relation`).
- `lineage_new_nodes` — דורות שהוזנו ידנית (`afterGeneration`, `name`, `relation`). נכנסים כ"ממתין לאישור" ומטופלים בצד האיגוד.

**תגובה — הצלחה:**
```json
{ "ok": true }
```

**תגובה — שגיאה:**
```json
{ "error": "<טקסט בעברית>" }
```

| Status | משמעות |
|--------|--------|
| `400` | שדה חובה חסר / ת"ז לא תקינה / כתובת חסרה / טלפון לא מאומת |
| `409` | ת"ז כבר קיימת במערכת |
| `429` | יותר מדי בקשות |

---

## זרימה מלאה (סיכום)

1. משתמש מקיש ת"ז → `GET /lookup`:
   - `found:true` **ללא** `is_child` → צאצא רשום: הצג `message` ועצור.
   - `found:true` **עם** `is_child:true` → ילד של צאצא: נתב לרישום מהיר (עם `parent_name`/`lineage_chain`).
   - `found:false` → המשך להרשמה הרגילה.
2. בחירת דורות: `GET /lineage-roots` → `GET /lineage-children?nodeId=...` דור אחר דור עד `children:[]`.
3. מילוי פרטים: `GET /cities` + `GET /streets?city=...`.
4. אימות טלפון: `POST /verify/send` → `POST /verify/confirm` → קבלת `token`.
5. שמירה: `POST /register` עם כל הפרטים + `token` + שרשרת הדורות.
