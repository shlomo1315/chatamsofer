## שלב 1 — הרץ את ה-migration ב-Supabase (🎯 ההשפעה הגדולה ביותר)

**קובץ:** `supabase/migrations/20260721_performance_indexes.sql`

מוסיף אינדקסים שחסרים על עמודות שמחפשים בהן — הופך full-table-scan לחיפוש אינדקס מיידי:
- **pg_trgm** על שם/משפחה/טלפון — חיפוש חופשי (`ILIKE '%טקסט%'`) שהיה סורק הכל
- **spouse_id_number** — חיפוש/בדיקת כפילות
- **activity_log(action, entity_id)** — הטבלה הכי צומחת

**הרצה:** Supabase → SQL Editor → הדבק את כל הקובץ → Run. (בטוח לחלוטין — אינדקסים בלבד, לא נוגע בנתונים.)

**בדיקה:**
```sql
select indexname from pg_indexes
where schemaname='public' and tablename='beneficiaries' and indexname like '%trgm%';
-- אמור להראות את אינדקסי ה-trigram החדשים
```

## שלב 2 (נפרד, יגיע בהמשך) — pagination + חיפוש בצד DB

הרשימות (נתמכים, הלוואות, יולדות, סיוע) עדיין טוענות את **כל** הטבלה לדף. באלפי
רשומות זה יאט את טעינת הדף. התיקון (pagination + חיפוש/סינון בצד ה-DB) הוא שינוי
ארכיטקטוני גדול יותר שייעשה ב-PR נפרד ומבוקר, אחרי ששלב 1 יציב.
